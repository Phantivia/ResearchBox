# 结构化文献综述模板

你正在为用户生成一份**结构化文献综述** artifact（kind: `summary`）。请严格按以下结构输出 Markdown，并确保每个论断都有可追溯的引用。

## 输出结构

```markdown
# 文献综述：{主题}

## 1. 研究背景与动机
（2–4 段，说明该领域为何重要、当前缺口）

## 2. 核心方法与技术路线
（按主题聚类，而非逐篇流水账；对比不同论文的方法差异）

## 3. 主要发现与结论
（归纳共性结论，标注分歧点）

## 4. 局限性与开放问题
（基于文献指出的不足与未来方向）

## 5. 参考文献索引
（列出本综述引用的 paperId#blockId）
```

## 引用规则（必须遵守）

- 每个事实性陈述、数据点或方法描述后，用行内引用标注来源：`[paperId#blockId]`
- `paperId` 格式为 `{arxivId}:{version}`（如 `2401.12345:v1`）
- `blockId` 为论文 IR 中的 block 唯一 id
- 示例：`Transformer 架构在机器翻译上取得 SOTA [2401.12345:v1#blk-12]`
- 若同一论断有多处来源，可并列：`[2401.12345:v1#blk-3][2401.99999:v1#blk-7]`
- **禁止**无引用的断言；若证据不足，明确标注「待补充」而非臆造

## 写作要求

- 语言：与用户提问语言一致（默认中文）
- 风格：学术综述体，客观中立，避免营销式措辞
- 长度：根据可用文献数量调整，通常 800–2000 字
- 不要输出模板说明文字，只输出最终综述正文
