ResearchBox Research Agent — P4 及之后阶段的需求规格
背景
ResearchBox 是纯前端、本地优先的学术论文阅读与研究工具。已完成的 Agent 基础设施:公共类型骨架、Agent 主循环(AsyncGenerator)、工具系统(自描述 Tool 接口)、人审批节点(human-in-the-loop)、权限模式(default / plan / autoApproveRead)。已有一个工具 paperbox_read(读单篇论文全文)。
论文以 PaperIR 形式存于 IndexedDB,每篇含结构化 blocks(带稳定 blockId)。"PaperBox(盒子)"指当前项目内已纳入的论文集合。
本规格定义接下来要建的工具、检索行为、以及"盒子开关"这一核心交互。

一、要新建/完善的工具
1. paperbox_list
列出盒内所有论文的标题、作者、摘要。供 agent 在检索前做导航,判断盒里已有什么、是否需要外部检索、该对哪几篇做深度检索。只读、可并行、低成本。
2. academic_search
学术文献搜索工具,以 arxiv 为中心。
数据源(分层降级):

默认:Semantic Scholar,无 API key 即可用。 所有未认证用户共享额度,够个人轻度使用,保证开箱即用。
可选增强:用户自配 API key。 OpenAlex 与 Semantic Scholar 都支持用户自带 key 获得更高、更稳定的额度。注意:OpenAlex 当前(2026 年)强制要求 API key,key 免费,用户需在 openalex.org/settings/api 注册获取;无 key 时 OpenAlex 不可用,回落到 Semantic Scholar。

技术约束:

Semantic Scholar 和 OpenAlex 均为纯 GET REST,浏览器可直接 fetch,无 CORS 问题。
arxiv 官方搜索 API 有 CORS 限制,不直接调用。

返回内容: 命中论文的 arxiv id + 标题 + 作者 + abstract。abstract 优先取搜索 API 返回值(Semantic Scholar 直接给纯文本;OpenAlex 给倒排索引,需重建为纯文本),搜索结果无 abstract 时再抓取 arxiv HTML 补齐。
纳入闸门(核心交互): 搜索结果不自动进盒。用户逐篇审查,决定是否将某篇加入 PaperBox。被纳入的论文才进入后续检索/研究的范围。这个"逐篇纳入"动作是外部文献进入盒子的唯一合法入口。
3. websearch
通用网页搜索,供 agent 获取论文之外的开放域信息。

用户自带 API key,内置 Tavily 与 Perplexity 两个适配器,二选一。
无免费方案:未配置 key 时该工具不注入工具池(或 fail-open 返回空,不影响 agent 其余运行)。
fail-open:网络失败/超时返回空结果,绝不让 agent 崩溃。

4. retrieval(封闭域检索,差异化核心)
对盒内论文做 RAG 式 block 级检索:接受 query,返回命中的 block(带 paperId#blockId 引用定位)。不上向量库,用轻量预筛 + 小模型 side-query 选区块的方案。
检索返回结果时,在给模型的消息中附带引导:命中相关 block 后,若需要该论文的完整上下文,可调用 paperbox_read(paperId) 拉取全文。

二、检索行为:两种姿态,由模型自主判断
文献检索没有 coding 那样可验证的二元收敛信号(测试通过 / 编译成功),它的收敛目标是 recall——"相关的是否都进来了"。但不是所有任务都要穷尽,姿态由任务意图决定:

探索式(explore): 快、广度优先、不求完整、几轮即收手。对应用户"看看这方向有些啥""随便搜搜"。优先响应速度。
穷尽式(exhaustive): 慢、追求召回饱和、需报告检索覆盖。对应用户"系统调研""做综述""meta-analysis 式文献搜罗"。

机制:

模型在系统提示层根据用户措辞自主选择姿态;不确定时默认探索式。
选择穷尽式时,模型可跟踪一个"已见 arxiv id / DOI 集合",当连续若干轮检索的新增命中趋近于零(召回饱和),向用户报告"检索已趋于饱和",作为收敛信号。这是模型可选的工作方式,不是硬编码进主循环的强制规则。
不要在主循环里硬编码任何 recall 数字阈值。 没有 ground truth,数字阈值是假的;真正的裁判是用户和饱和信号。
穷尽式任务收尾时,模型应能报告检索策略:用了哪些 query、覆盖了哪些来源、可能的盲区在哪(对标 meta-analysis 的可报告检索)。


三、盒子开关:采集与研究的边界
设计动机
这个产品要支持一条清晰的工作流:先从外部世界搜罗相关文献并纳入盒子,再纯粹基于盒内文献做回答与分析。 用户希望对"什么时候引入外部信息、什么时候只用盒内信息"有掌控感。
把这件事表达为"盒子的开关":

盒子开着(采集阶段): agent 可以用 websearch 和 academic_search 连接外部,目标是把相关文献拉进盒子。
盒子关着(研究阶段): 回答与分析绝对优先基于盒内文献。

标准工作流(必须保证全程连续、不打断)
打开应用(默认开盒)
  → academic_search / websearch 搜索
  → 逐篇审查,把相关论文纳入盒子
  → 循环上两步,直到用户认为文献搜罗收敛
  → 关盒
  → 基于盒内文献提问、研究、分析
这是一条单一连贯的叙事。设计上不得为了边界而打断它——不开新会话、不折叠对话历史、不清洗上下文。用户感知到的"关盒"只是点一下开关,然后继续提问,上下文完全连续。
关盒可以发生在流程的任意时刻,包括一个多步任务执行到一半。
边界的性质:引用优先级 + 透明度,不是访问控制
威胁模型很轻。学术来源(arxiv、OpenAlex、Semantic Scholar)本身具学术可信度,不是充斥虚假信息的随机网页;且用户全程看得见来源——每条外部结果在采集时都带来源标签显式出现过。因此边界不需要做成"对抗污染的硬隔离",而是做成"盒内优先 + 盒外兜底要透明标注"的软规则。这既给用户掌控感,又不付出隔离机制的工程与体验成本。
关盒的具体实现
盒子开关是一个会话级状态位(开 / 关)。关盒动作触发时,做两件事:
1. 向消息历史追加一条"边界标记(boundary marker)"消息。 内容是一条优先级规则,大意:

【盒子已关闭】从此标记起,回答必须绝对优先使用盒内论文内容,并以 paperId#blockId 形式引用。仅当盒内确实没有相关依据时,才可援引此前检索阶段获得的盒外信息;一旦援引盒外内容,必须明确标注"此点来自盒外、尚未正式纳入盒子",以便用户判断是否需要将相关文献正式纳入。

这条消息的作用是给模型一条只需服从的时间线规则(标记之前的外部内容降级为"检索过程记录",不作主要证据),而不是要求模型逐条回看历史、自行判断哪些内容是外部的——后者不可靠,前者可靠。它不删除任何历史,所以不破坏心流,也保留了"我是怎么找到这些论文的"这一叙事。
2. 系统提示中常驻的盒内优先规则配合生效。 不必在关盒时物理移除外部工具——靠系统提示约束"关盒后优先盒内、不主动外搜"即可。这样反方向也顺滑:研究中途想再补搜,重新开盒即可,无额外摩擦。
"盒内回答不了"时的行为
模型遇到盒内找不到依据的问题时:直接用盒外信息(若检索阶段获得过)给出兜底回答,并标注该点来自盒外、尚未纳入。是否把相关文献正式纳入盒子,由用户看到标注后自行决定。闸门(纳入决策)仍在用户手里,但是软提示而非硬阻断。
provenance 来源标记
每条进入上下文的信息带来源标签:paperbox(盒内论文)/ academic(学术搜索)/ web(网页搜索)。用途是给用户可见的透明标签,以及让模型在标注盒外引用时有据可依。

四、系统提示需包含的规则
在 agent 系统提示的稳定段加入:

检索姿态判断:探索式 vs 穷尽式的区分、判断依据(用户措辞)、默认探索式、不确定时主动询问。
穷尽任务的收敛标准:以 recall 充分性为准,收尾报告检索策略(query / 来源覆盖 / 盲区)。
盒内优先规则:盒子关闭后绝对优先盒内 paperId#blockId 内容;盒内缺失才用盒外兜底,且兜底必须标注来源为盒外。
引用规范:涉及论文内容的论断必须带 paperId#blockId 引用。


五、工作量画像

新功能(主要工作量): academic_search(含逐篇纳入闸门 + Semantic Scholar / OpenAlex 双适配器与 key 降级逻辑)、paperbox_list。
常规实现: websearch(Tavily / Perplexity 双适配器)、retrieval 及其拉全文引导。
状态与提示: 盒子开关状态位、boundary marker 注入、provenance 标签、上述系统提示规则。
检索姿态、召回饱和: 主要在系统提示层,饱和跟踪是模型可选的轻量工作记忆,非强制机制。
